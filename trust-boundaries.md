# Границы доверия и ответственности

Проект намеренно построен вокруг ограничений.

## Явно исключённые данные
- Государственные идентификаторы
- Закрытые и утёкшие базы
- Непубличные профили
- Покупные обогащающие данные

## Где автоматизация останавливается
- Сопоставление личности всегда вероятностное
- Конфликтующие сигналы не разрешаются автоматически
- Чувствительные сигналы требуют ручной проверки

## Запрещённые выводы
- Скоринг личности
- Рекомендации «нанять / не нанять»
- Психологическое профилирование

# Trust Boundaries

This project is intentionally defined by its limits.

## Explicitly excluded data
- Government-issued identifiers
- Private or leaked databases
- Non-public profiles
- Purchased enrichment data

## Where automation stops
- Identity matching is always probabilistic
- Conflicting signals are not auto-resolved
- Sensitive signals require human review
